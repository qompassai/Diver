Module learn_tables

Tables — Lua's only data structure, taught from the Diver config.

Lua has one container type: the table. It plays the role of array, map, struct, namespace, and module all at once. Almost every line of diver is building, merging, or iterating tables. Master this module and the config stops looking mysterious.

Functions

make_list ()
Make a list (1-based array) of plugin names.

When you write { a, b, c } with no keys, Lua assigns integer keys starting at 1. This is the "sequence" or "array part" of a table. diver's lua/utils/docs/init.lua does exactly this with its modules list, then loops it with ipairs.

Returns:

    string[]

Usage:

    local list = learn_tables.make_list(); print(list[1]) --> "bounty"
each (list, fn)
Walk a list in order with ipairs.

ipairs(t) yields index, value pairs from 1 upward and **stops at the first nil**. Use it whenever order matters. This mirrors the require-loop in lua/utils/docs/init.lua and lua/types/core/init.lua.

Parameters:

  • list any[]
  • fn fun(index: integer, value: any)

Usage:

    learn_tables.each(list, function(i, v) print(i, v) end)
keymap_opts (bufnr, desc)
Build the options table you would hand to vim.keymap.set.

When you write keys explicitly ({ noremap = true }) the table is acting as a map / record. This is the exact opts table built in lua/mappings/aimap.lua before each mapping. Note: desc powers which-key and :map listings.

Parameters:

  • bufnr integer
  • desc string

Returns:

    { noremap: boolean, silent: boolean, buffer: integer, desc: string }

Usage:

    local o = learn_tables.keymap_opts(0, "Toggle Explorer")
each_pair (map, fn)
Walk a map with pairs (order NOT guaranteed).

pairs(t) iterates every key, string or number, in an unspecified order. Use it for maps/records where order does not matter. diver uses it in schema.lua (for name, schema in pairs(opts.replace)).

Parameters:

  • map table
  • fn fun(key: any, value: any)
dig (root, path)
Safely read a deeply-nested field without crashing on a missing parent.

Indexing nil (a.b.c when a.b is nil) raises an error. diver avoids this with vim.tbl_get and with and-chains. Here we show the hand-rolled version so you understand what those helpers do under the hood.

Parameters:

  • root table
  • path string[]

Returns:

    any

Usage:

    local v = learn_tables.dig(client, {"server_capabilities", "hoverProvider"})
merge (base, overrides)
Merge a base options table with per-call overrides.

This is THE pattern behind vim.tbl_extend('force', opts, { desc = ... }) seen all over lua/mappings/. Later keys win. diver also uses the deep variant vim.tbl_deep_extend('force', M.config, config) in schema.lua's setup.

Parameters:

  • base table
  • overrides table

Returns:

    table

Usage:

    local o = learn_tables.merge({silent=true}, {desc="My map"})
module_convention ()
Explain why almost every diver file starts with local M = {}.

A Lua module is just a table you return at the bottom of the file. You attach public functions as function M.foo() end and keep helpers as local function bar() end. require('config.core.lsp') then hands the caller back exactly that M table. This function returns a description of the convention so it shows up in the generated docs.

Returns:

    string
with_defaults (opts)
Normalise an optional options table, applying defaults.

Public functions in diver accept a single opts table and start with opts = opts or {} so callers can omit it entirely (see M.config(opts) in lua/config/init.lua and simple_colon_parser in parser.lua). This makes APIs forgiving and future-proof.

Parameters:

  • opts ? { severity?: string, source?: string }

Returns:

    { severity: string, source: string }

Usage:

    learn_tables.with_defaults()                 --> {severity="WARN", source="diver"}
learn_tables.with_defaults({source="lint"})  --> {severity="WARN", source="lint"}
