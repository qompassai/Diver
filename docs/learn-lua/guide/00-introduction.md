# Learn Lua with Diver

Welcome. This course teaches you **Lua** — not in the abstract, but through the
exact code that powers the [Diver](https://github.com/qompassai/diver) Neovim
configuration. By the end you will be able to open any file in `diver`, read it,
explain what it does, and change it with confidence.

## Who this is for

You understand basic programming ideas — you know the difference between a
boolean and a float, you know what a variable and a function are — but you
cannot yet sit down and write Lua from scratch without leaning heavily on an AI
assistant. That is exactly the right starting point. The goal here is to get you
to the point where the config is *readable*, and small edits feel *safe*.

## How the course is built

This documentation is generated by **LDoc** (also called LuaDoc), the standard
documentation generator for Lua, using comments written in **LuaCATS**
annotation style. That choice is deliberate and is itself a lesson:

- **LDoc** turns specially-formatted comments (`--- @param`, `--- @return`) into
  the HTML pages you are reading now.
- **LuaCATS** is the annotation dialect understood by `lua-language-server`, the
  language server that gives Neovim its Lua hovers, autocompletion, and red
  error squiggles.

The remarkable part: **one comment satisfies both tools**. A line like

```lua
---@param bufnr integer the buffer number
```

is read by `lua-language-server` for type-checking *and* by LDoc for these docs.
`diver` already uses these annotations everywhere (look at any file in
`lua/types/`), so learning to read them is learning to read the config.

## How to study

Each chapter follows the same rhythm:

1. **Concept** — a short, plain-English explanation of one Lua idea.
2. **In Diver** — a pointer to the real file and line where you can see it.
3. **Reference** — the runnable, annotated example in the left-hand module list
   (`learn_lua`, `learn_tables`, `learn_vim_api`). Click "View source" to read
   the fully-commented Lua.

The fastest way to learn is to keep `diver` open in Neovim in one window and
this documentation in the other. Read a chapter, then jump to the cited file and
find the pattern with your own eyes.

## A note on versions

`diver` targets **Neovim 0.13** and runs on **LuaJIT** (which implements the
**Lua 5.1** language plus extensions). Everything in this course is written for
that combination. Where a feature is specific to the modern native API
(introduced across 0.11–0.13), the text calls it out explicitly. The headline
changes — `vim.pack`, the `lsp/*.lua` config layout, native completion, native
inline completion — are covered in Chapter 11.

Turn to **Chapter 01** when you are ready.
